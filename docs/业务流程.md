# 业务流程图

## 1. 老带新裂变主流程

```
┌────────────────────────────────────────────────────────────┐
│                  老带新裂变主流程                            │
└────────────────────────────────────────────────────────────┘

老客 A                    系统                    新客 B
  │                         │                         │
  │ ①打开小程序             │                         │
  ├───────────────→         │                         │
  │  生成专属推荐码          │                         │
  │←────qrCode─────────────│                         │
  │                         │                         │
  │ ②分享给好友             │                         │
  ├─────────────────────────┼────────────→            │
  │                         │                         │
  │                         │  ③扫码进入              │
  │                         │←──────────────          │
  │                         │  url?referrerId=A       │
  │                         │                         │
  │                         │  ④建立推荐关系          │
  │                         │  INSERT t_referral      │
  │                         │  (referrer=A, referred=B)│
  │                         │                         │
  │                         │  ⑤用户注册/登录         │
  │                         │  status=0 (新客)        │
  │                         │                         │
  │                         │  ⑥浏览商品下单          │
  │                         │  payAmount=¥50          │
  │                         │                         │
  │                         │  ⑦调用统一下单          │
  │                         │  → 微信/支付宝           │
  │                         │                         │
  │                         │  ⑧支付回调              │
  │                         │  pay-callback           │
  │                         │  校验签名 + 更新状态     │
  │                         │                         │
  │                         │  ⑨裂变结算              │
  │                         │  · B.status = 1 (老客)  │
  │                         │  · B 获 8.5 折券         │
  │                         │  · A 获 30 元免单券     │
  │                         │  · referral.status = 1  │
  │                         │                         │
  │ ⑩收到通知               │                         │
  │←───消息推送─────────────│                         │
  │ "你的好友B已首单,        │                         │
  │  送你30元免单券"        │                         │
  │                         │  ⑪后续消费              │
  │                         │  使用 30 元免单券       │
  │                         │  实际只付 20 元         │
```

## 2. 券生命周期

```
[生成]  --→  [未使用]  --→  [已使用]  --→  完结
   │            │  ↑
   │            ↓  │
   │        [已过期] ←─ expire_time < now
   │
   └─ 创建于: 首单回调 / 商家手动发券
```

## 3. 用户状态机

```
未注册 → 新客户(status=0)  ──首单支付成功──→  老客户(status=1)
                                  │
                                  ├─→ 触发裂变奖励
                                  │     ├─→ 给推荐人发免单券
                                  │     └─→ 给自己发折扣券
                                  │
                                  └─→ 状态升级
```

## 4. 支付时序

```
[小程序]                [后端]                [微信支付]
   │                      │                      │
   │ POST /order/create   │                      │
   ├──────────────────→   │                      │
   │   (amount, payType)  │                      │
   │                      │ 创建订单(未支付)      │
   │←orderId──────────────│                      │
   │                      │                      │
   │ POST /order/pay      │                      │
   ├──────────────────→   │                      │
   │   (orderId, openid)  │                      │
   │                      │ 统一下单              │
   │                      ├──────────────────→   │
   │                      │                      │
   │                      │←──prepay_id──────────│
   │                      │                      │
   │←{prepay_id}──────────│                      │
   │                      │                      │
   │ wx.requestPayment    │                      │
   ├──────────────────────┼──────────────────→   │
   │   (用户输入密码)      │                      │
   │                      │                      │
   │                      │  支付完成,异步通知    │
   │                      │←─────────────────────┤
   │                      │ POST /pay-callback   │
   │                      │  · 验签              │
   │                      │  · 更新订单状态      │
   │                      │  · 触发裂变结算      │
   │                      │                      │
   │                      │───响应 SUCCESS──→    │
   │                      │                      │
   │ 主动查询订单状态       │                      │
   │ GET /order/detail    │                      │
   ├──────────────────→   │                      │
   │← payStatus=1 ────────│                      │
   │                      │                      │
   │ 弹窗"支付成功"        │                      │
   │ 跳转订单详情          │                      │
```

## 5. 防刷机制

```
注册时
  ├─ Redis 检查: 同 phone 1 分钟内是否注册过 → 拒绝
  ├─ Redis 检查: 同 IP 1 分钟内是否超过 10 次 → 拒绝
  └─ 写库 + Redis 计数

发券时
  ├─ 查 user 当日发券数 >= 5 → 拒绝
  └─ 写券 + 异步通知

支付时
  ├─ 验签失败 → 拒绝
  ├─ 同 device 1 小时内多笔首单 → 人工审核
  └─ 金额异常(如 > 单店日均 10x) → 人工审核
```
