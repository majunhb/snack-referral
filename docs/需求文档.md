# 小吃店"老带新"裂变营销小程序 - 需求文档

> 基于《小吃店"老带新"裂变营销小程序解决方案》V1.0 拆解

## 1. 项目背景

传统小吃店面临获客难、推广成本高的痛点，社交裂变（"老带新"）模式是低成本获取新客户的高效手段。

**项目目标**：为小吃店搭建一套完整的"老带新"裂变营销系统，通过免单券+折扣券双向激励实现客户拉新。

## 2. 项目目标

- ✅ 微信 + 支付宝双平台小程序
- ✅ 老-新客户关系自动绑定
- ✅ 新客首单后双方自动发券
- ✅ 免单券封顶、限额等反作弊
- ✅ 形成可持续裂变闭环

## 3. 业务流程

```
┌────────────┐  扫码   ┌────────────┐
│ 老客 A     │────────→│ 新客 B     │
│ (老客户)   │         │ (新客户)   │
└────────────┘         └────────────┘
      │                       │
      │                       ▼
      │              ┌────────────────┐
      │              │ 注册/登录      │
      │              │ 绑定推荐人 A   │
      │              └────────────────┘
      │                       │
      │                       ▼
      │              ┌────────────────┐
      │              │ 下单 + 支付    │
      │              │ (首单)         │
      │              └────────────────┘
      │                       │
      │                       ▼
      │              ┌────────────────┐
      │              │ 支付回调       │
      │              │ 触发裂变奖励   │
      │              └────────────────┘
      │                       │
      ▼                       ▼
┌────────────────┐    ┌────────────────┐
│ 获 30 元免单券 │    │ 获 8.5 折券    │
│ (推人奖励)     │    │ (新客优惠)     │
└────────────────┘    └────────────────┘
```

## 4. 权益规则

| 类型 | 名称 | 规则 |
|---|---|---|
| 免单券 | 推荐奖励 | 单笔封顶 30 元，每日最多发 5 张，有效 30 天 |
| 折扣券 | 新客优惠 | 默认 8.5 折，有效 30 天 |
| 使用限制 | 通用 | 每单限用 1 张，不可叠加 |

## 5. 业务核心

### 5.1 推荐关系建立
- 老客 A 邀请新客 B 扫码进入
- 系统在 `t_referral` 表建立 (referrer=A, referred=B, status=0) 记录
- B 完成首单 → status=1，自动给 A 发 30 元免单券

### 5.2 首单判断
- 支付回调时检查 `t_user.status`：0=新客, 1=老客
- 升级逻辑: 新客首次支付成功 → status=1
- 关键点: 用 user.status 判定，避免并发问题

### 5.3 防刷机制
- 同手机号/设备/IP 频控
- 每日发券上限（默认 5 张）
- 免单券金额封顶（默认 30 元）
- 后台人工审核通道

## 6. 用户端功能

| 模块 | 功能 |
|---|---|
| 注册/登录 | 微信/支付宝一键授权 + 手机号绑定 |
| 首页 | 推荐状态展示（老/新客户标识）、可用券数、邀请入口 |
| 邀请页 | 生成专属二维码、保存到相册、好友分享、邀请战绩 |
| 权益页 | 4 个 Tab：全部/可使用/已使用/已过期 |
| 消费记录 | 订单列表、首单标识、用券标识 |
| 支付 | 微信/支付宝双通道、券自动计算 |

## 7. 商家端功能

| 模块 | 功能 |
|---|---|
| 数据看板 | 总用户/老客/新客、营收、首单数、ROI |
| 用户管理 | 用户列表、状态筛选、关键字搜索 |
| 订单管理 | 订单列表、首单标识、用券标识 |
| 权益管理 | 券列表、状态筛选 |
| 系统设置 | 免单券金额、折扣率、有效期、发券上限（实时生效） |

## 8. 数据表（共 6 张）

详见 `server/sql/schema.sql`：

1. `t_user` - 用户表
2. `t_coupon` - 权益券表
3. `t_order` - 订单表
4. `t_referral` - 推荐关系表
5. `t_config` - 系统配置表
6. `t_admin` - 商家管理员表
